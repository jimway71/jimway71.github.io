安装配置JupyterLab

在Window下安装配置JupyterLab

环境: Windows 10

安装Python3和pip3

已安装, 版本信息

c:\>python --version
Python 3.10.6
c:\>pip –version
pip 22.2.2 from C:\Users\username\AppData\Local\Programs\Python\Python310\lib\site-packages\pip (python 3.10)

建立Python虚拟环境

目标: 把JupyterLab的虚拟环境建到"d:\虚拟机\python\jlab_venv"目录下

  • “d:\虚拟机" 虚拟机目录
  • “d:\虚拟机\python" python虚拟环境目录
  • “d:\虚拟机\python\jlab_venv" JupyterLab的虚拟环境

c:\> d:
c:\> cd 虚拟机
c:\> mkdir python
c:\> cd python

建立python虚拟环境

c:\> python -m venv d:\虚拟机\python\jlab_venv

进入python虚拟环境

c:\> d:\虚拟机\python\jlab_venv\Scripts\activate.bat

在python虚拟环境中安装JupyterLab

在虚拟环境中安装jupyterlab

(jlab_venv) D:\虚拟机\python> pip install jupyterlab

在虚拟环境中安装中文语言包

(jlab_venv) D:\虚拟机\python> pip install jupyterlab-language-pack-zh-CN

在虚拟环境中安装JupyterLab Codex

(jlab_venv) D:\虚拟机\python> pip install jupyterlab-codex

在python虚拟环境中运行Jupyter Lab

(jlab_venv) D:\虚拟机\python> jupyter lab

正常启动后, 会自动使用浏览器访问JupyterLab

在Linux下安装配置JupyterLab

环境: Ubuntu 22.10 server, 未安装Xwindow

安装Python3和pip3

# (需要以root用户操作)安装Python3
# apt install python3
# 安装pip3
# apt install python3-pip

安装和配置python虚拟环境

参考:

  • Python中的虚拟环境（Virtualenv）及其工作原理

# (以root用户身份操作)安装python-venv(python虚拟环境)
# apt install python3.10-venv
# 以非root用户操作, 建立python虚拟环境
# 这是抄windows下的命令, Linux下命令应该是: virtualenv -p python3 ~/jlab_venv. 但windows命令也有效
$ python3 -m venv ~/jlab_venv
# 进入python虚拟环境
$ source ~/jlab_venv/bin/activate
(jlab_venv) user@host:~$

确认进入了python虚拟环境, 后面python操作都在python虚拟环境中执行

配置Nginx代理

  • JupyterLab运行于127.0.0.1:8888端口的处理方案一: 使用Nginx做反向代理
    Nginx做反向代理方案使JupyterLab运行不正常: 在ipynb中的python语句不能运行后显示结果
    通过对比测试, 确认了上述结论
    原因: 可能是未开户某些协议的支持

说明:

  • 缺省配置下, JupyterLab运行时会监听127.0.0.1:8888端口
    在本机能跑浏览器的情况下, 这完全没有问题
    但是, 在Ubuntu Server中, 缺省没有安装Xwindow
    • 没有Xwindow, 不能跑窗口界面
    • 没有窗口界面, 不能跑浏览器

而且,

  • JupyterLab所监听的127.0.0.1:8888端口, 也不能被其它同在同一网络的机器访问

因此, 这个端口需要被代理出来, 转换成在网络上可被访问
最适合的工具就是Nginx, 方法就是做反向代理

安装Nginx

# apt install nginx

配置Nginx前向代理

$ cat /etc/nginx/sites-enabled/default

server {
        listen 80 default_server;
        listen [::]:80 default_server;

        root /var/www/html;

        # Add index.php to the list if you are using PHP
        index index.html index.htm index.nginx-debian.html;

        server_name _;

        location / {
                proxy_pass http://127.0.0.1:8888;
        }
}
......

配置后, 重启nginx

# service nginx restart

即可，网络上可以用"http://主机名/“访问之后搭建的JupyterLab

安装和运行JupyterLab

(在python虚拟环境中操作)

# 需要以非root用户操作, 安装JupyterLab
$ pip install jupyterlab
# (以非root用户操作)启动JupyterLab(后台运行)
$ jupyter lab &

运行开始后, 在目录”~/.local/share/jupyter/runtime"下会生成运行的一些记录文件, 文件名中的数字是jupyter的运行进程PID

  • JupyterLab运行于127.0.0.1:8888端口的处理方案二: 修改运行参数

  • 修改jupyter lab运行参数, 使其在0.0.0.0上监听
    (在python虚拟环境中操作)
    查看jupyter lab参数:

    $ jupyter lab --help
    …
    --ip=
    The IP address the Jupyter server will listen on.
    Default: ’localhost’
    Equivalent to: [--ServerApp.ip]
    --port=
    The port the server will listen on (env: JUPYTER_PORT).
    Default: 0
    Equivalent to: [--ServerApp.port]
    …

    运行命令:

    $ jupyter lab --ip=0.0.0.0 &

    即改为在0.0.0.0:8888上监听

配置中文界面

JypterLab语言包:

  • jupyterlab language-packs 多语言支持
    • jupyterlab language-packs list 支持的语言包列表

(在python虚拟环境中操作)
以非root用户身份运行

$ pip install jupyterlab-language-pack-zh-CN

即可完成安装JypterLab中文语言包
重启JupyterLab，通过"Settings→Language"更改语言为中文, 即可切换为中文界面

安装JupyterLab Codex

(在python虚拟环境中操作)
以非root用户身份运行

$ pip install jupyterlab-codex

安装完成后, 可以在Jupyter Lab的web界面上的插件管理器里看到这个插件
