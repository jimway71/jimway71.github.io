原因:

  • 根据"Pycharm 2020.3 - Internal Error - java.net.BindException: Address already in use: bind - Windows 10“中的说法:

    To lock folders IDE is starting a server on localhost, it tries to bind on the first available port between 6942 and 6991, this exception is thrown if IDE was not able to bind on any of the ports in this range.

    PyCharm启动时, 需要从tcp端口6942到6991中找到一个可用的端口, 如果找不到, 就会报这个错误。

关于Systemd任务类型、任务文件目录

  • 由于Systemd要做的事情太多，导致单元有很多不同的种类，大概一共有12种。举例来说，Service 单元负责后台服务，Timer 单元负责定时器，Slice 单元负责资源的分配。
    每个单元都有一个单元描述文件，它们分散在三个目录。
    • /lib/systemd/system：系统默认的单元文件
    • /etc/systemd/system：用户安装的软件的单元文件
    • /usr/lib/systemd/system：用户自己定义的单元文件

Systemd定时器相关文件包括

  • Service单元(文件, 文件名后缀.service, 比如certbot.service)
  • Timer单元(文件, 文件名后缀.timer, 比如certbot.timer)

其中

  • Service单元就是所要执行的任务
    文件内容格式如下:

    # cat certbot.service
    [Unit]
    Description=Certbot
    Documentation=file:///usr/share/doc/python-certbot-doc/html/index.html
    Documentation=https://letsencrypt.readthedocs.io/en/latest/
    [Service]
    Type=oneshot
    ExecStart=/usr/bin/certbot -q renew

  • Service单元定义了如何执行任务
    要定时执行这个Service，还必须定义Timer单元
    文件内容格式如下:

    # cat certbot.timer
    [Unit]
    Description=Run certbot twice daily

    [Timer]
    OnCalendar=--* 00,12:00:00
    RandomizedDelaySec=43200
    Persistent=true

    [Install]
    WantedBy=timers.target

    其中, [Timer]部分定制定时器。Systemd 提供以下一些字段

    • OnActiveSec：定时器生效后，多少时间开始执行任务
    • OnBootSec：系统启动后，多少时间开始执行任务
    • OnStartupSec：Systemd 进程启动后，多少时间开始执行任务
    • OnUnitActiveSec：该单元上次执行后，等多少时间再次执行
    • OnUnitInactiveSec： 定时器上次关闭后多少时间，再次执行
    • OnCalendar：基于绝对时间，而不是相对时间执行
    • AccuracySec：如果因为各种原因，任务必须推迟执行，推迟的最大秒数，默认是60秒
    • Unit：真正要执行的任务，默认是同名的带有.service后缀的单元
    • Persistent：如果设置了该字段，即使定时器到时没有启动，也会自动执行相应的单元
    • WakeSystem：如果系统休眠，是否自动唤醒系统

下面的命令可以查看所有的单元文件。

# 查看所有单元
$ systemctl list-unit-files

# 查看所有 Service 单元
$ systemctl list-unit-files --type service

# 查看所有 Timer 单元
$ systemctl list-unit-files --type timer
# 或
$ systemctl list-timers --all

以下是定时器的相关命令:

# 启动mytimer.timer定时器
$ sudo systemctl start mytimer.timer

# 查看mytimer.timer定时器的状态
$ systemctl status mytimer.timer

# 查看所有正在运行的定时器
$ systemctl list-timers

# 关闭mytimer.timer定时器
$ sudo systemctl stop myscript.timer

# 下次开机，自动运行这个定时器
$ sudo systemctl enable myscript.timer

# 关闭定时器的开机自启动
$ sudo systemctl disable myscript.timer

另外, Systemd的日志功能很强，提供统一的命令, 如下:

# 查看整个日志
$ sudo journalctl

# 查看 mytimer.timer 的日志
$ sudo journalctl -u mytimer.timer

# 查看 mytimer.timer 和 mytimer.service 的日志
$ sudo journalctl -u mytimer

# 从结尾开始查看最新日志
$ sudo journalctl -f

# 从结尾开始查看 mytimer.timer 的日志
$ journalctl -f -u timer.timer

    • shellinabox_2.21build1_amd64.deb版本, 改为http协议后可用

  • 在Armbian Linux 5.3.0-aml-g12(Ubuntu 18.04.3 LTS)上, 已确认
    • shellinabox_2.21_arm64.deb版本 可用(需改设置)

所测原生apt安装包都可使用, 只需将协议由"https"改为"http”:
修改/etc/default/shellinabox, 参数"SHELLINABOX_ARGS"改为(增加"-t"参数):

SHELLINABOX_ARGS="--no-beep -t"

即协议由原"https", 改为"http"后, 确认可以正常访问并使用。
考虑到shellinabox都会由Nginx反向代理, 所以使用http协议没有问题。

修改shellinabox的配置
参考: Shell in a Box - Remote to your Linux Server via Web Browser
Ubuntu上shellinabox的配置文件是: /etc/default/shellinaboxd, 在该文件中

  • SHELLINABOX_PORT 监听端口
  • SHELLINABOX_ARGS 运行参数
    运行参数改为"--no-beep -t", 即改为使用http协议, 即改为访问"http://地址:端口"(缺省安装为用https协议)
    运行参数改为"--no-beep -t --localhost-only", 即改为使用http协议监听127.0.0.1, 即改为访问"http://localhost:端口"(缺省安装为用https协议, 监听0.0.0.0)

Vmware workstation 17设置自动启动报错 Failed

报错信息如下:

  • 无法更新"自动启动"配置。
    请确保存在vmAutoStart.xml文件, 并且您有权写入此文件。

参考:

  • 为 Windows 主机上的本地虚拟机启用自动启动 来自Vmware官方文库

需要确认以下几点

  • 该虚拟机配置文件(.vmx, 在该虚拟机文件目录下)的所有者(或称属主) 为 用户A
  • 在指定目录(当前环境下为C:\ProgramData\VMware\VMware Workstation目录)下存在vmAutoStart.xml文件, 且用户A有写权限
  • 已安装VMware自动启动服务
  • 在VMware自动启动服务中, 设置以用户A的身份执行操作
  • 启动VMware自动启动服务

操作:
虚拟机名为testas

  • 虚拟机配置文件testas.vmx所有者(或称属主) 为 userA
  • 修改"C:\ProgramData\VMware\VMware Workstation\vmAutoStart.xml"文件属性, 在"安全"里, 点击编辑, 并为userA添加写入权限
  • 运行"services.msc", 并确认到已安装了VMware自动启动服务
  • VMware自动启动服务的"属性"-“登录"中, 设置以userA身份登录
  • 设置完成后启动VMware自动启动服务
  • 即可回到Vmware workstation中去设置虚拟机随系统自动启动

结果:
设置虚拟机随系统自动启动操作成功完成, 但是, 经过测试, 并未能实现vmware虚拟机跟随windows系统启动

怎样下载"知乎"网站上的视频

参考: 怎么下载知乎上的视频？

其实只需要一个谷歌浏览器（火狐的操作方法一样）就可以，无需任何插件，也不需要去关注微信公众号。
任何谷歌浏览器内核都可以用以下方法：
1，用谷歌浏览器点一个有视频的回答。
2，等待视频打开，在视频画面中右键点“检查”。
3，右方点“network”，然后按“F5”刷新。

然后在方框中输入vzuu，vzuu为知乎视频网站的主机名，下方就会过滤出链接来。

第三个和第四个就是显示为media，在这两个链接任意右键点拷贝链接，下载软件下载即可。
如果不想用下载软件下载，把复制好的视频网址在浏览器中打开，然后右上角三个点-更多工具-网页另存为，就可以用谷歌浏览器下载。
很多网站都可以用这个方法下载，可以用media分辨。
