● Critical Internal Error on Startup of IntelliJ IDEA: “Cannot Lock System Folders”
●Unable to bind ports: Docker-for-Windows & Hyper-V excluding but not using important port ranges #3171
● The default dynamic port range for TCP/IP has changed since Windows Vista and in Windows Server 2008
● Pycharm 2020.3 - Internal Error - java.net.BindException: Address already in use: bind - Windows 10

原因:

  • 根据"Pycharm 2020.3 - Internal Error - java.net.BindException: Address already in use: bind - Windows 10“中的说法:

    To lock folders IDE is starting a server on localhost, it tries to bind on the first available port between 6942 and 6991, this exception is thrown if IDE was not able to bind on any of the ports in this range.

    PyCharm启动时, 需要从tcp端口6942到6991中找到一个可用的端口, 如果找不到, 就会报这个错误。

记录:
其实也不知道哪条命令起的作用:
● 先停了Hyper-V中的全部虚拟机
● 把tcp动态端口范围调整到25192-300, 重启后, PyCharm就可以正常启动了;

—————- Hugo服务器相关问题 —————-

  • 问题:
    Hugo服务器并未支持html标签。
    现象:
    在Hugo+theme-stack上, 发现html标签并未生效。 例如: 下划线标记未生效
1

<u>下划线</u>

显示为(如果正常显示, 说明已正确设置):

下划线

查看网页源代码显示为:

1
2
3

<blockquote>
<p><!-- raw HTML omitted -->下划线<!-- raw HTML omitted --></p>
</blockquote>

解决方式:
修改config.yaml, 修改相关部分内容如下:

1
2
3
4
5

markup:
    goldmark:
        renderer:
            ## Set to true if you have HTML content inside Markdown
            unsafe: true

config.yaml这部分说明中, ‘## Set to true if you have HTML content inside Markdown’, 已经说明需要设置unsafe为true, 以使html标签生效
