在R3300L类+Armbian上安装Hugo服务器

记录: 在R3300L类+Armbian上安装Hugo服务器

Aug 08, 2022
阅读时长: 11 分钟

在R3300L(及类似架构设备)等使用S905系列的电视盒子上安装Armbian, 在Armbian系统下可以安装Hugo服务器。

编译及安装Hugo

选择: Hugo最新版本, +extended, 支持linux/arm64系统

  • 说明: R3300L的CPU是S905L, Arm架构CPU, 四核A53

有多种方式可以安装Hugo, Armbian提供的apt工具就可以安装Hugo, 但基于Ubuntu传统, 使用这种方式下载的版本较旧, 很多功能并不支持。
注: 当前Armbian上使用apt可以安装的Hugo版本仅为0.40.1-1, 如下:

1
2
3
4
5

$ apt search hugo
Sorting... Done
Full Text Search... Done
hugo/bionic 0.40.1-1 arm64
  Fast and flexible Static Site Generator written in Go

Hugo当前最新Release版本是v0.101.0, 下载地址: Hugo v0.101.0
但是, Release中仅提供了未+Extended的Linux/Arm64版本, 而缺少扩展功能(Extended), 有些功能或主题就无法使用，因此需要重新编译。

注意:

  • 不能使用手机+Termux, 借助手机较强的CPU性能和较高的配置进行编译, 因为编译出来的是For Android/Arm64, 而不是For Linux/Arm64, 因而无法使用。
  • 同理, 在电脑+Linux上做编译也不行。

编译hugo相关参考文档如下:

  • Build from Source on Mac
  • Install Update Hugo Extended for ARM in Ubuntu

编译要求:

  • >=Go 1.18版本的go
  • 最新版本的Hugo源代码包
  • (实际使用)2.8G的内存(含SWAP分区2G), 估计至少需要1G的内存(含SWAP分区)。
    否则会出现报错, 且运行极长的时间(以十数小时数计), 但是最后还无法成功完成编译。
    编译报错信息为:

    go build github.com/gohugoio/locales compressed:
    : /usr/local/go/pkg/tool/linux_amd64/compile: signal: killed

    参考:
    • Golang - go build编译项目时出现signal: killed
    • 用SWAP分区’ 解决golang编译项目时出现signal: killed’
  • /tmp空间需要大于400M, 估计至少应该超过1G: 否则会报错。
    编译报错信息为:

    * Github.com/bep/golibsass/internal/libsass
    /tmp/cctAMSUf.s: Assembler messages:
    /tmp/cctAMSUf.s: Fatal error: can’t write 3916 types to section
    .debug_loc of $WORK/b490/_x032.o: ‘No space left on device’
    /tmp/cctAMSUf.s: Fatal error: can’t close $WORK/b490/_x032.o: No space left on device

    在运行go build时查看/tmp目录, 确认编译时是使用/tmp做为编译目录, 也基本确认/tmp空间不足会出现错误。

编译操作:

  • 安装(当前)最新稳定版本的go
    参考: 初学GO笔记（1）-Arm64 linux上的安装。
    同样由于Ubuntu传统, 可以用apt直接安装的go包版本仅为1.10.3, 无法用于编译Hugo, 因此改用tar包安装。
    go包下载列表在go download, 当前最新稳定版本tar包是go1.18.4.linux-arm64.tar.gz:

    # wget https://go.dev/dl/go1.18.4.linux-arm64.tar.gz

    解压下载的go包:

    # tar zxvf go1.18.4.linux-arm64.tar.gz

    移到/usr/local下

    # mv go /usr/local/

    查看go的版本信息:

    # /usr/local/go/bin/go version
    go version go1.18.4 linux/arm64

    确认go版本没问题后, 开始建立go环境:

    # mkdir /opt/go
    # vim /etc/profile

    在文件末尾增加:

    export GOROOT=/usr/local/go
    export GOPATH=/opt/go
    export PATH=$GOPATH/bin:$GOROOT/bin:$PATH

    重新登录(新登录一回就行)后, 查看环境变量是否生效:

    # echo $PATH
    /opt/go/bin:/usr/local/go/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games
    # echo $GOROOT
    /usr/local/go
    # echo $GOPATH
    /opt/go
    确认环境变量已生效!
    修改配置, 改用国内仓库以提高速度:
    # go env -w GOPROXY=https://goproxy.cn,direct
    查看, 并确认已修改成功:
    # go env | grep GOPROXY
    GOPROXY=“https://goproxy.cn,direct”

    原为: GOPROXY="https://proxy.golang.org,direct"
    确认Go1.18.4已安装完成

  • 在R3300L上准备编译Hugo的环境

    • 准备内存空间, 以配合编译Hugo的内存需求(估计至少大于1G)
      由于R3300L的实际内存仅为805M, 无法满足Hugo编译需要, 因此需要临时增加Swap分区以增加可用内存。
      创建做为swap分区的文件, 命令中创建了一个2G的文件:

      # dd if=/dev/zero of=/root/swapfile bs=1M count=2048

      格式化为交换分区文件, 建立swap的文件系统:

      # mkswap /root/swapfile

      启用交换分区(Swap)文件:

      # swapon /root/swapfile

      查看交换分区状态:

      # swapon
      NAME TYPE SIZE USED PRIO
      /dev/zram1 partition 400.5M 166.7M 5
      /root/swapfile file 2G 7.6M -2

      注意: 这样的设置方法, 系统重启后不再有效!
    • 增加/tmp目录空间, 以配合编译Hugo的存储空间需求(估计至少也大于1G)
      使用df命令查看空间:

      # df
      Filesystem 1K-blocks Used Available Use% Mounted on
      …
      tmpfs 410140 8 410132 1% /tmp
      …

      可以看到, /tmp的全部空间是400M, 确认需要临时增加/tmp的空间。 由于/root的空间够用, 因此改为/root下目录做为/tmp:
      新建tmp的目录:

      # mkdir /root/tmp

      给这个目录设置0777权限(避免有什么权限问题):

      # chmod 0777 /root/tmp

      把新建的/root/tmp目录bind到/tmp:

      # mount –bind /root/tmp /tmp

      再用df命令查看, 已看不到/tmp这一条记录。
      操作方法参考:
      • linux 修改/tmp目录的大小
      • mount –bind 的妙用
      注: 这样的设置方法, 系统重启后也不再有效

  • 编译Hugo
    Hugo源代码包的地址如下:
    • v0.101.0.zip
    或
    • v0.101.0.zip
    解压Hugo源代码tar包:

    # tar zxvf v0.101.0.zip

    进入Hugo目录:

    # cd v0.101.0

    开始编译Hugo, 带extended参数:

    # go build –tags extended

    编译完成只生成一个文件: hugo
    查看版本:

    # ./hugo version
    hugo v0.101.0+extended linux/amd64 BuildDate=unknown

    注: 在R3300L上两次成功编译Hugo的时长分别是: 24分钟, 3分钟, 很奇怪, 似乎都是这个规律。

themes分析:

  • docPort主题:
    适合用一个文档做整个网站入口, 即网站入口就是一个文档, 而不是一个页面(分别指向不同文档), 并不适合做普通网站, 比如博客(blog)
  • Pure主题:
    适合做blog:
    • 分左中右三列布局
    • 左/右有文本导航
    • 有标签(tags)/目录(Categories)分类
    • 左侧导航可以添加link链接
    • 左侧有多种配色可选
    • 左上标题、说明均可自定义
      补充: 后面选的Stack主题也满足上述要求

选择主题, 启动hugo

选择Pure主题, Pure相关说明文档:

  • Pure主题项目文档位于themes\pure目录下的README.MD(英文)+README-ZH.MD(中文)
  • IconFont图标说明文档位于: demo_fontclass
    相关的icon文件应该是位于themes\pure\static\fonts下的文件, 尤其是iconfont.svg这个文件

其他

已解决问题:

  • Pure主题的config.yml文件中, menu: -> main:中的图标来源不明(其中每一项的文字前都自动加上了图标)
    说明:
    图标的设置同样在config.yml文件中, 在menuIcons:后进行设置, 与menu->link->name对应, 格式如下:

    name: [这里有个空格]icon-name

    上述icon-name查demo_fontclass中的英文名称
  • Stack主题在左边导航链接添加Home(主页)的问题:
    如果按ExampleSite中的做法, 在\content\page里+home目录和md文件配置(仿照已有文件), md文件里title的名字应该写"主页";
    但这样配置后, 当使用浏览器访问网站主页时, 网站主页的名称(在浏览器左上角显示的网站名称)就会变成"主页", 而不是原来在hugo配置文件config.yaml配置里的title名称。
    处理:
    修改hugo配置文件config.yaml, 在menu:main:中增加Home, 适当配置weight(相当于相对位置的值), 可以得到期望的效果。
    修改配置文件blog/config.yaml, 如下:

menu:
    main: 
        - identifier: Home
          name: 主页
          url: /
          params:

  • Stack主题调整左侧导航栏文字及图标(icon)的颜色问题。
    说明:
    hugo主题配置中, themes下的配置信息会被网站根目录下相应的配置信息所替代,
    如blog\themes\hugo-theme-stack\assets\scss\中的配置信息, 可以被blog\assets\sccs\下的配置替代。
    这样, 修改配置时, 不需要修改themes目录下的主题原始配置文件, 而是仅在网站目录下进行修改, 而且只设置需要修改的字段即可 - 简单、方便, 且便于代码管理。
    左侧导航栏文字原配置的颜色太浅, 需要修改。

    增加目录blog\assets\sccs, 并将原blog\themes\hugo-theme-stack\assets\scss\custom.scss文件(基本是个空文件, 用于用户自定义的配置)拷贝到该目录下, 在该custome.scss中添加如下配置:

其他问题

关于在Windows10上, 使用TortoiseGit(+Git)管理在Samba服务器(基于Armbian系统)上的仓库时出现的问题

问题描述:
在Armbian系统上使用git clone命令克隆Pure主题仓库(相关目录通过Samba服务器在网络上共享), 在Armbian系统中和Windows共享中修改后, 在Windows上使用TortoiseGit查看diff, 报"fatal: detected dubious ownership in repository"类的错误, 并提示:

  • 添加环境变量GIT_TEST_DEBUG_UNSAFE_DIRECTORIES=true
  • 使用 git config --global --add safe.directory '仓库目录全路径' 命令, 应该是添加信任目录

其中:

  • 环境变量可以使用windows的"编辑系统环境变量"工具添加, 在cmd中使用set命令查看确认
  • 添加信任目录可以在cmd中使用已安装的git.exe程序执行

但是, 按此操作后, 并没有效果: 使用TortoiseGit查看diff依然报错。
另外, 在内存虚拟磁盘(使用’Olof Lagerkvist’的’ImDisk Virtual Disk’建立)上, 也会出错同样的问题。

处理:
在windows本机磁盘上clone仓库, 并同样进行修改, 可以正常使用git进行操作。

结论:
在windows上, 暂时无法使用git工具操作Samba共享目录上的仓库, 可能与权限相关, 因为用户大概率不同 - samba上的文件owner应该是Armbian系统中的用户。
