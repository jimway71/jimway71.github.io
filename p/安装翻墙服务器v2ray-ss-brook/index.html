在Ubuntu18上安装翻墙服务器(迁移)

简单说明:
系统: Ubuntu 18.04 LTS。
a.安装: V2RAY mKCP+dtls, AAAA; ss: BBBB; brook: CCCC。
b.(V2ray安装的)shadowsocks改用了新的加密方式(AHEAD: chacha20-ietf-poly1305), 以避免被GFW检测到
终端软件上, 不再使用单独的shadowsocks终端, 而改用v2ray终端(包含shadowsocks终端)
电脑上使用V2rayN, 安卓手机上使用BifrostV

在Ubuntu Server 18.04版上进行安装

0.确认unzip已安装(可以直接运行unzip)
如果没有安装, 用下面命令装(如果unzip没有安装, 会自动提示这条命令)

# sudo apt install unzip

1.V2RAY+ss(单端口)安装, 再+BBR
V2RAY一键安装
选择:
V2RAY ~ 协议 mKCP_dtls, 端口 AAAA
ss 和原来一样
https://ssr.tools/1026 安装时, BBR自己就装上了

使用233版V2ray一键安装脚本(集成BBR/锐速/Shadowsocks):

# bash <(curl -s -L https://git.io/v2ray.sh)

2.Brook安装
参考
https://doub.io/brook-jc1/

1)下载

# mkdir /root/brook/
# cd /root/brook/

最新版本地址: https://github.com/txthinking/brook#download

# wget -N –no-check-certificate “https://github.com/txthinking/brook/releases/download/v20200201/brook"
# chmod +x brook

2)启动

# nohup ./brook server -l :CCCC-p passwd > /dev/null 2>&1 &

3)防火墙
[ 现在没开云服务器ubuntu自身的防火墙iptable/ufw

# sudo ufw allow CCCC

]

云的防火墙也要打开

4)开机自启动
由于ubuntu 18.04的变化, 原rc.local初始不再有效, 需要处理, 参考:
Ubuntu 18.04 rc.local systemd设置
https://blog.csdn.net/zhengchaooo/article/details/80202599

# ln -fs /lib/systemd/system/rc-local.service /etc/systemd/system/rc-local.service
# touch /etc/rc.local
# chmod 755 /etc/rc.local

并在/etc/rc.local第一行加上下面这行:

#!/bin/bash

后面一样
参考
http://blog.csdn.net/fireroll/article/details/15503463

放在/etc/rc.local里
加入:

nohup /root/brook/brook server -l :CCCC-p passwd > /dev/null 2>&1 &

注意brook所在目录

  2. 初始化
    用SecureCRT(或其它类似软件)用ssh连接服务器
    1) apt更新
        # apt update
        # 先安装vim
        # apt install vim
    2) 安装ufw防火墙, 并设置规则及自动启动
         # 安装ufw防火墙: 已缺省安装
         # 设置规则
         # ufw allow 22/tcp; ufw allow 22222/tcp; ufw allow 443/tcp
 
       注: 22222做为新的ssh端口, 免的被人反复试密码
        # 启动ufw防火墙, 并查看确认ufw状态
        # ufw enable; ufw status

	Status: active
	To                         Action      From
	--                         ------      ----
	22/tcp                     ALLOW       Anywhere                  
	22222/tcp                  ALLOW       Anywhere                  
	443/tcp                    ALLOW       Anywhere                  
	22/tcp (v6)                ALLOW       Anywhere (v6)             
	22222/tcp (v6)             ALLOW       Anywhere (v6)             
	443/tcp (v6)               ALLOW       Anywhere (v6) 

        # 设置跟随系统启动
        # systemctl enable ufw


    3) 修改ssh端口为22222
        # vim  /etc/ssh/sshd_config
        修改如下:
        # Port 22
        Port 22222
        # 保存后, 重启sshd
        # service sshd restart
        用secruecrt连接22222端口, 测试确认新的端口生效

    4) 重新设置用户
        3个用户: root / userA /none
           root不直接登录
           userA登录转su做操作
           none 不能转su, 现在用于做sshesame(伪装ssh服务器的启动用户身份)
       # 增加用户userA
       # adduser userA
       # 注 - 密码: (先设为XXXXXXXX)
       # 设置为可转su用户
       # usermod -aG sudo userA
       -  测试 , 用userA用户登录, 执行: $ sudo whoami


       # 修改root密码
       # passwd
       # 写一复杂密码, 同样不用记录

    # cat /etc/crontab 
        # /etc/crontab: system-wide crontab
        # Unlike any other crontab you don't have to run the `crontab'
        # command to install the new version when you edit this file
        # and files in /etc/cron.d. These files also have username fields,
        # that none of the other crontabs do.

        SHELL=/bin/sh
        PATH=/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin

        # 增加这行, 设置cron所用的时区为中国(已确认可用)
        CRON_TZ=Asia/Shanghai

        # m h dom mon dow user  command
        17 *    * * *   root    cd / && run-parts --report /etc/cron.hourly
        25 6    * * *   root    test -x /usr/sbin/anacron || ( cd / && run-parts --report /etc/cron.daily )
        47 6    * * 7   root    test -x /usr/sbin/anacron || ( cd / && run-parts --report /etc/cron.weekly )
        52 6    1 * *   root    test -x /usr/sbin/anacron || ( cd / && run-parts --report /etc/cron.monthly )
        # 由于设置了CRON_TZ为中国, 这里的时间是写中国时间.
        # 经测试可用
        # 每周二/四/六/日(中国时间)早上5点自动重启
         10 04 * * 2,4,6,0 root reboot
